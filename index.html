Introdución

Os sistemas de control de versións son unha categoría de ferramentas de software que axudan a rexistrar os cambios realizados nos ficheiros mantendo un control das modificacións feitas no código. 

Un produto de software está normalmente desenvolvido por un grupo de desenvolvedores que poden estar situados en diferentes lugares. Cada compoñente do grupo pode ter que facer modificacións no código fonte.

Os sistemas de control de versións axudan a xestionar de forma eficiente todos os cambios que se van facendo no código fonte xunto coa información de quen o fixo e cando se produciu a modificación.

A dinámica xeral de traballo con un sistema de control de versión consistirá en:

  • A rama principal conterá o código fonte principal.
  • Cada colaborador creará a súa rama para o seu traballo
  • Só cando o traballo colaborativo estea validado, pódese fusionar co código fonte principal.
